## Prompt korekcyjny B — globalny undo/redo stosu geometrii

```

You are working on the 4dach-qt application (PyQt/PySide, Python).
Branch: feat/freehand-fusion360-like
Files to modify: ui/drawing_canvas.py (primary),
ui/main_window.py (only to wire Ctrl+Z / Ctrl+Y if needed)

AppSettings already has `undo_stack_depth: int = 50` but DrawingCanvas
has NO geometry undo/redo stack. There is only `_sketch_redo_points`

Your task is to add a proper geometry undo/redo stack for ALL
post-draw geometry mutations. No refactoring, no new features beyond

---

## What must be undoable (the undo stack)

Every operation that mutates the finalized geometry of an existing
polygon must push an undo record BEFORE applying the change:

1. Dragging a vertex of the outline polygon.
2. Dragging an edge midpoint handle (move_vertices mode OR
insert_vertex mode — both produce a new polygon state).
3. Dragging the entire outline polygon (move body).
4. Dragging a vertex of a cutout (hole) polygon.
5. Dragging the entire cutout polygon (move hole body).
6. Editing an edge length label inline (confirming with Enter).
7. Editing a vertex angle label inline (confirming with Enter).
8. Dragging the origin point (when origin edit is enabled).

Closing a new polygon (polygon_closed / cutout_closed signal) is
handled by the controller ABOVE the canvas, so do NOT add it to
the canvas undo stack — the canvas only owns edits to already-
existing polygons.

---

## Implementation requirements

### 1. UndoRecord dataclass

Define a private dataclass inside drawing_canvas.py:

```python
@dataclass(slots=True)
class _UndoRecord:
    operation: str          # e.g. "drag_vertex", "drag_body", "label_edit", ...
    outline_before: Polygon2D
    holes_before: list[Polygon2D]
    origin_before: Point2D | None
```

"Before" = the state BEFORE the operation.

### 2. Undo/redo stacks as instance variables

Add to `DrawingCanvas.__init__`:

```python
self._undo_stack: list[_UndoRecord] = []
self._redo_stack: list[_UndoRecord] = []
```

Max depth is `self._app_settings.undo_stack_depth` (default 50).
When pushing to `_undo_stack`, if `len(_undo_stack) > max_depth`,
pop from the front (oldest entry).
Any mutation push MUST clear `_redo_stack`.

### 3. Helper methods

```python
def _push_undo(self, operation: str) -> None:
    """Snapshot current geometry and push to undo stack."""

def _undo(self) -> None:
    """Pop from undo stack, restore geometry, push to redo stack."""

def _redo(self) -> None:
    """Pop from redo stack, restore geometry, push to undo stack."""
```

`_push_undo` must:

- Read `self.roof_plane.outline` (or the active preview outline if
different) and `self.roof_plane.holes` at the moment of the call.
- Make deep copies (do NOT store references — store new Polygon2D
instances with copied points lists).

`_undo` and `_redo` must:

- Restore `self.roof_plane.outline` and `self.roof_plane.holes`
from the snapshot.
- Emit `outline_edit_committed` / `hole_edit_committed` signals
as appropriate so the controller above persists the restored state.
- Reset all drag state and preview state.
- Call `self.update()`.


### 4. Where to call _push_undo

Call `_push_undo(operation)` at the START of drag (on mouse-press
that initiates a drag), NOT at the end. This way the "before"
snapshot is always captured before any change.

For inline label edits, call `_push_undo("label_edit")`
in `_confirm_post_draw_editor()` BEFORE applying the new geometry.

Specific insertion points:

- `_start_outline_drag()` → `self._push_undo("drag_vertex")`
- `_start_outline_edge_split_drag()` → `self._push_undo("drag_edge")`
- `_start_plane_body_drag()` → `self._push_undo("drag_body")`
- `_start_hole_drag()` → `self._push_undo("drag_hole_vertex")`
- `_start_hole_center_drag()` (or equivalent) → `self._push_undo("drag_hole_body")`
- `_confirm_post_draw_editor()` → `self._push_undo("label_edit")`

If any of these method names differ slightly in the actual code,
find the correct method by its behavior (initiates a geometry drag
or confirms a label edit) and insert there.

### 5. Keyboard shortcuts

In `keyPressEvent`:

- Ctrl+Z → `self._undo()` (only when NOT in active freehand drawing
mode, since Ctrl+Z during drawing already does sketch undo via
`_sketch_redo_points`).
- Ctrl+Y or Ctrl+Shift+Z → `self._redo()`.

Sketch undo during drawing (`_sketch_redo_points`) must remain
unchanged — only trigger it when `self._mode in  {MODE_DRAW_OUTLINE, MODE_DRAW_CUTOUT}`.

Geometry undo (`_undo()`) triggers only when `self._mode in  {MODE_VIEW, MODE_EDIT, MODE_MOVE, MODE_IDLE}`.

### 6. Clear on new roof plane

In `set_roof_plane()`, after clearing drag state, also clear both stacks:

```python
self._undo_stack.clear()
self._redo_stack.clear()
```
