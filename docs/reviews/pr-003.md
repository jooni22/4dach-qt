> 🔴 **Signal connection accumulates on repeated draw-outline invocations, causing duplicate planes**
>
> Each call to `_start_draw_outline` connects `canvas.polygon_closed` to `self._on_polygon_closed` at line 396, but the connection is only disconnected inside `_on_polygon_closed` (line 402). If the user cancels drawing (via Escape or right-click) and starts drawing again, the signal accumulates multiple connections to the same canvas. When the polygon is finally closed, `_on_polygon_closed` fires once per connection. After the first invocation sets the outline on the active plane, each subsequent invocation calls `_set_active_plane_geometry` → `_ensure_plane_ready_for_geometry` (`ui/main_window.py:242-251`), which sees the plane now has an outline and creates a **new** empty plane, then sets its outline too — resulting in duplicate planes from a single user drawing action.
