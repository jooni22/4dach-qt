> **<sub><sub>![P1 Badge](https://img.shields.io/badge/P1-orange?style=flat)</sub></sub>  Derive slab span from one cross-section**
>
> The slab bounds are built with `min(top samples)` and `max(bottom samples)`, which mixes values from different `x` positions and can create a `raw_length_cm` that is larger than any real vertical section of the slab. On skewed quadrilaterals (for example a sheared rectangle/parallelogram with constant true strip height), this inflates sheet length and area, and can incorrectly trigger `exceeds_max_length` warnings/transverse split. Compute the slab span from the same sampled section (or from the maximum sampled local span) instead of combining independent extrema.
>
